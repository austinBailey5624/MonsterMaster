Classes
Name Purpose
Actor Interface that Monsters and PlayerActors implement - it holds basic 'fighting' information: hp mp level etc
Battlefield Holds two arrays of Actors -Allies and enemies- and pushes to a queue that alternates between them, checks if an actor is dead and removes it from the queue, handles the menus associated with the battle
Bestiary Acts as the 'pokedex': Holds and organizes the basic information about the MonsterTypes, reads the names and descriptions from file
Damage It is the class that holds all of the different damage types, basically a wrapper for two arrays- the actual damage and the status damage
Equippable This a class that extends the Items class along with (eventually) QuestItem and Useable- these Items are the Items that the player character can equip to increase their combat effectiveness in various ways
Item This is an interface that is extended by the Equippable, QuestItem and Useable classes
List Reader This class handles reading the information pertaining to the Objects from file -all of the names and descriptions for the Bestiary class for example
List Writer This class is responsible for writing the files that the List Reader reads - most of the methods are only used once but I keep it around anyway just incase
Main Basically its a menu system that calls other functions. It also has methods that are used across a lot of functions, like getRandomNumber and verify() (which is used to verify input to functions)
Monster This class Inherits from Actor and handles each individual monster
MonsterType Used to generate what type (or species) the monster is I.E. bulbasaur
Party Holds four Actors, and handles the methods involved with equipping, looking at stats, etc
Place Basically a two dimensional 'field' of Terrains that may or may not be transversed across
PlayerCharacter Extends Actor, Houses information specific to the playerCharacter
Skill It is a constructor for the skills that will be used in the game- 1 for each monsterType plus some others
Terrain A terrain is basically a single square of a place that can be transversible or not like a wall or a path. Stuff can also happen when you transverse across them, like monster battles for instance
Useable Useable extends Item and these are items that can be used during battle or from the menu
